Backend development rules:

1. Before every backend task, read docs/06_后端接口与数据逻辑.md and docs/后端当前任务.md.
2. Only complete one backend task at a time.
3. Before modifying code, explain which files will be changed.
4. Do not change existing API behavior without explanation.
5. Do not change frontend code.
6. Do not change the question JSON structure unless the current task clearly requires it.
7. Do not do large-scale refactoring.
8. If a task changes API, schema, model, database, or business rules, update docs/06_后端接口与数据逻辑.md.
9. After finishing each task, append a record to docs/后端开发日志.md.
10. Do not overwrite old development logs.
11. Always provide API test steps.
12. If something cannot be implemented because of missing database fields or unclear logic, mark it clearly as a limitation or TODO.
