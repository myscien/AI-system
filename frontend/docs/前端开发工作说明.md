# 前端开发工作说明

项目名称：基于 Dify 工作流的课末巩固练习生成与学情反馈系统

前端定位：面向课堂课末 5 分钟即时评价场景，承担学生答题、即时反馈、教师任务管理、题目导入、学生名单导入、学情分析展示与教学摘要复制等交互功能。

## 1. 前端技术栈

本项目采用轻量化 Vue 前端技术栈：

- Vue 3：使用单文件组件 `.vue` 组织页面。
- Vue Router：实现首页、答题页、学生学习详情页、教师任务页、教师分析页之间的路由跳转。
- Vite：作为开发与构建工具。
- JavaScript 原生 `fetch`：封装接口请求，集中放在 `src/api/tasks.js`。
- 组件化页面结构：当前主要按页面组件组织，页面文件位于 `src/pages/`。
- 本地状态与浏览器存储：使用 `ref`、`reactive`、`computed`、`watch` 管理状态；使用 `localStorage` 保存学生学号、学生信息和答题进度。
- CSS 响应式布局：统一样式在 `src/style.css` 中维护，支持移动端折叠布局。

当前项目未引入 UI 组件库，页面控件、表格、卡片、状态标签、进度条等均为自定义实现。

## 2. 页面与功能模块

当前路由结构在 `src/router/index.js` 中定义，主要页面如下：

- `/`：`HomePage.vue`
- `/practice/:taskId`：`PracticePage.vue`
- `/student/tasks/:taskId/summary`：`StudentTaskSummaryPage.vue`
- `/teacher/tasks`：`TeacherTasksPage.vue`
- `/teacher/analysis/:taskId`：`TeacherAnalysisPage.vue`

### 学生端已实现功能

- 学生入口/学号识别  
  在首页输入 8 位学号，调用 `getTasks(studentNo)` 获取学生信息与任务列表。

- 任务列表  
  首页展示当前学生可做任务，包括任务标题、年级、题目数、完成进度、正确率、最近提交时间等。

- 学生答题页面  
  `PracticePage.vue` 根据任务 ID 加载题目，支持单选题、多选题、判断题、填空题。

- 单题提交  
  学生每道题点击“提交本题”后，前端先本地判定正确性，再调用 `submitSingleQuestion(taskId, questionPk, payload)` 保存过程性作答记录。

- 即时反馈  
  答题后页面通过选项颜色、选项解释、提示文案、解析区域展示即时反馈。

- 再次尝试  
  第一次答错后，系统清空当前选择，提示“你还可以再尝试一次”，允许学生再次提交。

- 查看答案  
  两次答错后显示“查看答案”按钮，点击后展示正确答案和解析，并将该题标记为完成。

- 鼓励语展示  
  题目下方保留鼓励卡片，根据首次答对、首次答错、订正成功、再次错误等状态展示不同鼓励语。顶部弹窗式鼓励语已移除，避免分散学生注意力。

- 练习完成总结  
  全部题目完成后展示本次练习总结，包括完成题数、首次答对数、订正后掌握数、查看答案数、不熟悉知识点和下一轮目标。

- 学生学习详情  
  `StudentTaskSummaryPage.vue` 展示单个任务的历史尝试记录、最近一次练习情况和题目结果。

### 教师端已实现功能

- 教师任务管理  
  `TeacherTasksPage.vue` 顶部展示任务列表，可刷新、导入题目、查看分析。

- 创建任务  
  支持输入标题、年级、说明，调用 `createTeacherTask(payload)` 创建任务。

- 题目导入  
  支持通过任务下拉选择 `id + title`，也保留手动输入任务 ID；粘贴 JSON 后调用 `importTeacherQuestions(taskId, questions)` 导入题目。

- 学生名单 CSV 导入  
  支持选择 `.csv` 文件上传，调用 `importTeacherStudentsCsv(file)`；前端不解析 CSV，只做文件类型和空文件校验，并展示读取总数、新增人数、更新人数。

- 学情分析展示  
  `TeacherAnalysisPage.vue` 支持按任务、年级、班级筛选学情数据。任务选择为 `id + title` 下拉；班级支持全部班级、1班、2班、3班、4班。

- 详细题目统计  
  展示题号、题干、题型、难度、知识点、正确率、作答次数、正确次数、错误次数、正确答案和选项选择次数。选择题/判断题显示选项统计条，填空题显示无选项统计状态。

- 学生完成情况表  
  展示学号、姓名、年级、班级、已答题数、正确数、正确率、完成进度、状态；支持状态筛选和排序。

- 教学摘要复制  
  调用 `/api/tasks/{taskId}/teaching-summary` 获取后端生成的 `copy_text`，可选择“不整合模板”“模板1：标准教学分析”“模板2：教研级分析”，复制给大模型使用。前端不重新拼学生姓名，直接使用后端已脱敏的 `copy_text`。

## 3. 核心交互逻辑

### 学生进入任务流程

1. 学生在首页输入 8 位学号。
2. 前端校验学号格式。
3. 调用 `/api/student/tasks?student_id=xxx` 获取学生信息和任务列表。
4. 学生点击“开始答题”进入 `/practice/:taskId?student_no=xxx`。
5. 前端将学生学号和学生信息保存到 `localStorage`，便于答题页恢复上下文。

### 逐题作答逻辑

1. 答题页调用 `/api/tasks/{taskId}/questions` 获取题目。
2. 前端为每道题建立本地状态，包括已选答案、首次答案、首次是否正确、提交次数、是否查看答案、是否完成等。
3. 单选/判断题使用 radio，多选题使用 checkbox，填空题使用输入框。
4. 点击“提交本题”后，本地先判断答案是否正确。
5. 随后调用 `/api/tasks/{taskId}/questions/{questionId}/submit` 保存单题作答记录。

### 答对后的反馈

- 标记本题完成。
- 选中选项显示正确样式。
- 可进入下一题。
- 显示题目下方鼓励卡片。
- 首次答对时解析默认折叠，避免打断答题节奏。

### 第一次答错后的反馈

- 记录首次错误答案。
- 展示错误选项反馈。
- 清空当前选择，允许学生重新作答。
- 显示“你还可以再尝试一次”。
- 不立即暴露完整答案，保留学生二次思考空间。

### 两次答错后的反馈

- 显示“查看答案”按钮。
- 点击后展示正确答案和完整解析。
- 本题标记完成，可进入下一题。
- 在最终总结中统计“查看答案”的题目数量。

### 教师查看分析流程

1. 教师进入 `/teacher/tasks` 查看任务列表。
2. 点击某个任务的“查看分析”，进入 `/teacher/analysis/:taskId`。
3. 教师可选择任务、年级、班级后点击“查询分析”。
4. 前端调用 `/api/tasks/{taskId}/analysis?grade=...&class_name=...`。
5. 页面展示概览指标、薄弱题目、详细题目统计、学生完成情况。
6. 教师可点击“复制教学摘要”，调用教学摘要接口并复制 `copy_text`。

## 4. AI 辅助开发证据

结合当前代码，以下功能适合体现“借助生成式人工智能辅助开发”的过程证据：

- 页面结构设计  
  项目形成了学生端、教师端、分析端的清晰页面拆分，路由结构明确，适合作为 AI 辅助规划页面结构的证据。

- 状态逻辑设计  
  `PracticePage.vue` 中包含较完整的答题状态机：首次提交、再次尝试、查看答案、完成状态、保存状态、错误状态、总结状态。这类复杂交互逻辑适合体现 AI 辅助梳理状态流转。

- 即时反馈策略优化  
  当前代码不仅判断对错，还区分首次答对、首次答错、订正成功、再次错误，并配合选项解释、解析展示和鼓励语。可体现 AI 辅助设计“低挫败感反馈机制”。

- 教师分析页面优化  
  学情分析页支持筛选、题目统计、选项统计、学生状态筛选排序、教学摘要复制。多个数据结构被前端归一化处理，适合体现 AI 辅助进行数据字段对接和页面改造。

- 接口封装与兼容处理  
  `src/api/tasks.js` 对后端字段进行了中英文/驼峰/下划线兼容归一化，如 `task_id/taskId/id`、`class_name/className` 等，适合体现 AI 辅助处理接口变更。

- 样式与交互细节优化  
  包括任务卡片、学生状态标签、题目统计条、响应式布局、标题点击回首页等，适合体现 AI 辅助进行前端体验迭代。

需要如实说明：当前代码中没有直接调用 Dify API。Dify 工作流更可能属于后端或外部生成流程，前端主要负责展示、答题、提交、分析和复制教学摘要。

## 5. 教学价值

本前端设计服务于“课末 5 分钟即时评价”的课堂场景，教学价值主要体现在：

- 降低学生操作门槛  
  学生只需输入 8 位学号即可获取任务，不需要复杂登录流程，适合课堂快速使用。

- 提升课末练习参与度  
  任务列表直接展示进度和正确率，学生能清楚知道自己当前任务状态。

- 支持即时反馈  
  学生每题提交后立即看到选项反馈、提示和解析，不需要等教师统一批改。

- 鼓励再次思考  
  第一次答错后不直接给答案，而是允许再次尝试，有利于学生纠错和二次思考。

- 降低答错挫败感  
  鼓励语以题目下方卡片形式展示，避免弹窗干扰，同时保留积极反馈。

- 帮助学生形成复盘意识  
  完成后总结首次答对、订正后掌握、查看答案、不熟悉知识点和下一轮目标，帮助学生知道自己哪里需要巩固。

- 帮助教师快速掌握课堂情况  
  教师端可以查看完成率、平均正确率、低正确率学生、薄弱题目和选项错误分布，便于课后或下一节课调整教学。

- 支持教师生成教学建议材料  
  教学摘要复制功能可将脱敏后的课堂数据复制给大模型，辅助教师生成教学反思、改进建议或教研材料。

## 前端开发工作摘要

本项目前端基于 Vue 3、Vue Router 和 Vite 构建，面向信息科技课堂课末 5 分钟巩固练习场景，实现了学生学号识别、任务列表、逐题作答、单题提交、即时反馈、再次尝试、查看答案和练习总结等功能。教师端支持任务管理、题目 JSON 导入、学生 CSV 名单导入、按任务/年级/班级查看学情分析，并展示薄弱题目、详细题目统计、选项选择次数和学生完成状态。前端还支持复制后端生成的脱敏教学摘要，并可选择整合教学分析模板，便于教师借助大模型生成教学建议。整体设计降低了课堂使用门槛，增强了学生即时反馈体验，也帮助教师快速掌握练习效果。
