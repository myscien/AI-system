# teaching-summary 接口前端对接说明

本文档只说明教学摘要接口，供前端开发“生成教学摘要 / 复制教学摘要”功能使用。

## 1. 接口地址

```http
GET /api/tasks/{task_id}/teaching-summary
```

示例：

```http
GET /api/tasks/1/teaching-summary
GET /api/tasks/1/teaching-summary?grade=6&class_name=1
```

## 2. 查询参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `grade` | string | 否 | 年级，例如 `6`、`7`、`8` |
| `class_name` | string | 否 | 班级存储值，例如 `1` |

规则：

- 不传 `grade` 时，后端默认使用任务自己的年级。
- 传 `class_name` 时，后端按班级过滤学生数据。
- 参数应与当前学情分析页选择的任务、年级、班级保持一致。

前端调用示例：

```js
const params = new URLSearchParams()

if (grade) params.set("grade", grade)
if (className) params.set("class_name", className)

const query = params.toString()
const url = query
  ? `/api/tasks/${taskId}/teaching-summary?${query}`
  : `/api/tasks/${taskId}/teaching-summary`

const res = await fetch(url)
const data = await res.json()
```

## 3. 接口用途

该接口用于生成结构化教学摘要，帮助教师：

1. 查看当前任务、年级、班级范围下的整体学习情况。
2. 查看薄弱题目、题干、选项、正确答案和主要错误选项。
3. 查看薄弱知识点。
4. 查看学生分层情况。
5. 一键复制 `copy_text`，用于给大模型生成教学建议。

## 4. 返回结构总览

```json
{
  "task": {
    "task_id": 1,
    "task_title": "六年级分数练习",
    "grade": "6",
    "class_name": "1"
  },
  "overview": {},
  "weak_questions": [],
  "knowledge_points": [],
  "student_groups": {},
  "copy_text": "..."
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `task` | object | 当前教学摘要对应的任务和筛选范围 |
| `overview` | object | 整体情况 |
| `weak_questions` | array | 薄弱题目列表 |
| `knowledge_points` | array | 知识点统计 |
| `student_groups` | object | 学生分层 |
| `copy_text` | string | 可直接复制的教学摘要文本 |

  "task_id": 1,

## 6. overview

示例：

```json
{
  "assigned_students": 48,
  "submitted_students": 42,
  "not_started_students": 6,
  "completed_students": 36,
  "completion_rate": 0.75,
  "average_accuracy": 0.68,
  "low_accuracy_student_count": 12
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `assigned_students` | number | 当前筛选范围内应参与学生数 |
| `submitted_students` | number | 已提交学生数 |
| `not_started_students` | number | 未开始学生数 |
| `completed_students` | number | 已完成学生数 |
| `completion_rate` | number | 完成率，0 到 1 |
| `average_accuracy` | number | 平均正确率，0 到 1 |
| `low_accuracy_student_count` | number | 低正确率学生数 |

前端展示百分比时需要乘以 100。

## 7. weak_questions

`weak_questions` 包含薄弱题目的题干、选项、正确答案、选择人数和主要错误选项。

示例：

```json
{
  "question_id": 3,
  "question_pk": 3,
  "knowledge_point": "分数乘法",
  "question_type": "single_choice",
  "difficulty": "medium",
  "stem": "下列计算正确的是？",
  "options": [
    {
      "id": "A",
      "text": "1/2 × 1/3 = 1/6",
      "is_correct": true,
      "selected_count": 18
    },
    {
      "id": "B",
      "text": "1/2 × 1/3 = 1/5",
      "is_correct": false,
      "selected_count": 16
    }
  ],
  "correct_answer": ["A"],
  "total_answers": 42,
  "correct_count": 18,
  "wrong_count": 24,
  "accuracy": 0.4286,
  "top_wrong_options": [
    {
      "option_id": "B",
      "option_text": "1/2 × 1/3 = 1/5",
      "count": 16
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `question_id` | number | 题目 ID |
| `question_pk` | number | 题目主键 |
| `knowledge_point` | string | 知识点 |
| `question_type` | string | 题型 |
| `difficulty` | string | 难度 |
| `stem` | string | 题干 |
| `options` | array | 选项列表 |
| `correct_answer` | string[] | 正确答案 |
| `total_answers` | number | 当前范围内作答人数 |
| `correct_count` | number | 答对人数 |
| `wrong_count` | number | 答错人数 |
| `accuracy` | number | 正确率，0 到 1 |
| `top_wrong_options` | array | 主要错误选项 |

薄弱题规则：

- 优先返回正确率低于 `0.6` 的题目。
- 如果没有低于 `0.6` 的题，返回有作答记录且正确率最低的前 3 题。

## 11. student_groups

示例：

```json
{
  "not_started": [],
  "in_progress": [],
  "low_accuracy": [],
  "high_accuracy": []
}
```

分组说明：

| 分组 | 说明 |
|---|---|
| `not_started` | 未开始学生 |
| `in_progress` | 进行中学生 |
| `low_accuracy` | 已作答且正确率低于 `0.6` 的学生 |
| `high_accuracy` | 已作答且正确率大于等于 `0.85` 的学生 |

学生对象结构：

```json
{
  "student_no": "20300101",
  "student_name": "张三",
  "grade": "6",
  "class_name": "1",
  "answered_count": 8,
  "correct_count": 4,
  "accuracy": 0.5,
  "progress": 0.8,
  "status": "in_progress"
}
```

注意：

- JSON 中 `student_name` 保留完整姓名，供教师端系统内展示。
- `copy_text` 中学生姓名已经脱敏，适合复制给外部大模型。

## 12. copy_text

`copy_text` 是一段可以直接复制的教学摘要文本。

用途：

- 前端做“复制教学摘要”按钮。
- 老师可以复制给大模型，让大模型生成教学建议。
- 老师也可以直接阅读。

示例片段：

```text
任务：六年级分数练习
范围：6年级1班

一、整体情况
应参与 48 人，已提交 42 人，未开始 6 人，已完成 36 人。
完成率 75.0%，平均正确率 68.0%，低正确率学生 12 人。

三、薄弱题目
1. 题目ID：3
知识点：分数乘法
题型：single_choice
难度：medium
正确率：42.9%，作答人数：42，错误人数：24
题干：
下列计算正确的是？
选项：
A. 1/2 × 1/3 = 1/6（正确，18人选择）
B. 1/2 × 1/3 = 1/5（错误，16人选择）
正确答案：A
主要错误选项：B 1/2 × 1/3 = 1/5，16人选择

四、学生分层
未开始学生：6 人
进行中学生：8 人
低正确率学生：12 人
高正确率学生：10 人
低正确率学生名单示例：
- 姓名：张** 学号：20300101 正确率：50.0%
```

隐私要求：

- `copy_text` 中学生姓名已脱敏。
- 脱敏格式为：`姓名：张** 学号：20300101`。
- 前端复制时直接复制 `copy_text`，不要自己重新拼接学生姓名。

## 13. 前端功能要求

前端需要实现：

1. 在教师学情分析页增加“生成教学摘要”或“复制教学摘要”按钮。
2. 点击按钮时调用 `/api/tasks/{task_id}/teaching-summary`。
3. 请求参数使用当前页面已选择的 `task_id`、`grade`、`class_name`。
4. 接口返回后，复制 `copy_text` 到剪贴板。
5. 可选：在页面上展示 `overview`、`weak_questions`、`knowledge_points`、`student_groups`。
6. 若 `weak_questions` 为空，展示“暂无明显薄弱题目”。
7. 若接口返回错误，展示后端返回的 `detail`。

复制示例：

```js
await navigator.clipboard.writeText(data.copy_text)
```

## 14. 错误处理

任务不存在：

```json
{
  "detail": "Task not found"
}
```

前端处理建议：

- HTTP `404`：提示任务不存在或已删除。
- HTTP 非 `2xx`：读取 `detail` 并展示。
- `copy_text` 为空：禁用复制按钮或提示暂无摘要。
