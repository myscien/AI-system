# 05_前端页面与状态逻辑

本文档基于当前 `frontend` 目录下真实代码整理，最后核对时间为 `2026-04-24`。

## 1. 前端技术栈和启动方式

- 框架：Vue 3，使用 `<script setup>`
- 构建工具：Vite 5
- 路由：Vue Router 4
- HTTP：浏览器原生 `fetch`
- API 封装文件：`src/api/tasks.js`
- 路由文件：`src/router/index.js`
- 页面文件目录：`src/pages/`
- 根入口：`src/main.js`
- 根组件：`src/App.vue`

启动方式：

- 安装依赖：`npm install`
- 本地开发：`npm run dev`
- 构建：`npm run build`
- 预览：`npm run preview`

开发环境下，`/api` 由 Vite 代理到后端服务。

## 3. 当前路由结构

当前使用 Vue Router 管理页面。

路由表：

1. `/`
   - 学生入口与任务列表页
   - 页面文件：`src/pages/HomePage.vue`

2. `/practice/:taskId`
   - 学生答题页
   - 页面文件：`src/pages/PracticePage.vue`
   - 约定从 query 读取 `student_no`
   - 示例：`/practice/3?student_no=20280101`

3. `/teacher/analysis/:taskId`
   - 教师学情分析页
   - 页面文件：`src/pages/TeacherAnalysisPage.vue`
   - 示例：`/teacher/analysis/3`

4. `/teacher/tasks`
   - 教师任务管理与题目导入页
   - 页面文件：`src/pages/TeacherTasksPage.vue`

页面流转关系：

- `/ -> /practice/:taskId?student_no=...`
- `/ -> /teacher/tasks`
- `/teacher/tasks -> /teacher/analysis/:taskId`

当前入口策略：

- 教师统一先从 `/teacher/tasks` 进入
- 首页不再直接跳转到分析页

## 5. 学生点击“提交本题”的当前执行流程

当前按钮逻辑在 `src/pages/PracticePage.vue` 的 `submitCurrentQuestion()` 中。

执行流程：

1. 调用 `evaluateCurrentAnswer()` 在前端做本地判题
2. 未作答时写入 `error` 并终止
3. 更新当前题状态：
   - `submitCount`
   - `currentIsCorrect`
   - `feedbackAnswer`
   - `feedbackOptionExplanation`
   - `studentExplanation`
4. 首次提交时记录：
   - `firstSubmittedAnswer`
   - `firstIsCorrect`
5. 若本次答对，则设置 `completed = true`
6. 调用 `persistQuizState()` 写入 `localStorage`
7. 调用 `saveQuestionResult(questionId, selectedAnswer)` 实时保存到后端
8. `saveQuestionResult()` 内部规则：
   - 当前题本会话已提交成功，则不再请求后端
   - 当前题正在提交中，则不再重复提交
   - 成功后写入 `submittedQuestionMap`
   - 失败后写入 `saveErrorMap`

## 6. 当前教师任务管理页状态与逻辑

教师任务页定义在 `src/pages/TeacherTasksPage.vue`。

核心状态：

- `teacherTasks`
- `loadingTasks`
- `creatingTask`
- `importingQuestions`
- `listErrorMessage`
- `createErrorMessage`
- `createSuccessMessage`
- `importErrorMessage`
- `importSuccessMessage`
- `createForm`
  - `title`
  - `grade`
  - `description`
- `importForm`
  - `taskId`
  - `jsonText`

教师任务页执行流程：

1. 进入 `/teacher/tasks`
2. 页面加载时调用 `getTeacherTasks()`
3. 创建任务时：
   - 校验标题与年级非空
   - 调用 `createTeacherTask()`
   - 成功后清空表单并刷新任务列表
4. 导入题目时：
   - 校验任务 ID
   - 校验 JSON 非空
   - 校验 JSON 合法
   - 校验对象中包含非空 `questions` 数组
   - 校验通过后，将 `questions` 数组提交给后端
   - 成功后刷新任务列表并跳转到分析页
5. 任务列表操作：
   - “导入题目”按钮会把当前任务 ID 填入导入表单
   - “查看分析”按钮跳转到 `/teacher/analysis/:taskId`

当前限制：

- 题目导入前端要求粘贴的 JSON 结构为 `{ "questions": [...] }`
- 发送给后端时，实际提交的是其中的 `questions` 数组
- 任务列表中的 `created_at` 当前后端未返回，前端显示为 `-`

## 8. 当前前端调用的后端 API

当前 API 封装都在 `src/api/tasks.js`。

学生侧：

1. `GET /api/student/tasks?student_id={studentNo}`
   - 方法：`getTasks(studentNo)`

2. `GET /api/tasks/{taskId}/questions`
   - 方法：`getQuestions(taskId)`

3. `POST /api/tasks/{taskId}/questions/{questionId}/submit`
   - 方法：`submitSingleQuestion(taskId, questionId, payload)`

4. `POST /api/tasks/{taskId}/submit`
   - 方法：`submitAnswers(taskId, payload)`

教师侧：

5. `GET /api/tasks/{taskId}/analysis`
   - 方法：`getTaskAnalysis(taskId)`

6. `GET /api/teacher/tasks`
   - 方法：`getTeacherTasks()`

7. `POST /api/teacher/tasks`
   - 方法：`createTeacherTask(payload)`

8. `POST /api/teacher/tasks/{taskId}/questions/import`
   - 方法：`importTeacherQuestions(taskId, questions)`
