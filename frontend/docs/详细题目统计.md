# 详细题目统计前端对接说明

本文说明前端在统计分析页需要对接的接口、筛选参数、返回结构和展示任务。

## 1. 前端本次任务

前端需要在任务统计分析页支持：

1. 按年级查看统计数据。
2. 按班级查看统计数据。
3. 展示每道题的题目信息。
4. 展示每道选择题、判断题的选项选择次数。
5. 展示学生完成情况，包含未开始答题的学生。

后端已经在原有分析接口上扩展，不需要新增前端接口地址。

## 2. 分析接口

接口地址：

```http
GET /api/tasks/{task_id}/analysis
```

示例：

```http
GET /api/tasks/1/analysis
GET /api/tasks/1/analysis?grade=6
GET /api/tasks/1/analysis?grade=6&class_name=1
```

## 3. 查询参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `grade` | string | 否 | 年级，例如 `6`、`7`、`8` |
| `class_name` | string | 否 | 班级存储值，例如 `1` |

规则：

- 不传 `grade` 时，后端默认使用当前任务的 `task.grade`。
- 传 `grade` 时，后端按传入年级过滤学生。
- 传 `class_name` 时，后端在年级范围内继续按班级过滤。
- `class_name` 必须和学生名单导入时保存的班级文本一致。

前端调用示例：

```js
const params = new URLSearchParams()

if (grade) params.set("grade", grade)
if (className) params.set("class_name", className)

const query = params.toString()
const url = query
  ? `/api/tasks/${taskId}/analysis?${query}`
  : `/api/tasks/${taskId}/analysis`

const res = await fetch(url)
const data = await res.json()
```

## 4. 返回结构总览

成功返回：

```json
{
  "task_id": 1,
  "task_title": "六年级分数练习",
  "total_students": 48,
  "assigned_students": 48,
  "submitted_students": 42,
  "completed_students": 36,
  "completion_rate": 0.75,
  "average_accuracy": 0.8125,
  "low_accuracy_student_count": 5,
  "question_stats": [],
  "student_progress": [],
  "lowest_accuracy_questions": []
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `task_id` | number | 任务 ID |
| `task_title` | string | 任务标题 |
| `total_students` | number | 当前筛选范围内学生总数 |
| `assigned_students` | number | 当前筛选范围内 active 学生总数 |
| `submitted_students` | number | 当前筛选范围内有答题记录的学生数 |
| `completed_students` | number | 当前筛选范围内完成全部题目的学生数 |
| `completion_rate` | number | 完成率，`completed_students / assigned_students` |
| `average_accuracy` | number | 当前筛选范围内总体平均正确率 |
| `low_accuracy_student_count` | number | 低正确率学生数 |
| `question_stats` | array | 每道题的详情和统计 |
| `student_progress` | array | 每个学生的完成情况 |
| `lowest_accuracy_questions` | array | 正确率最低的题目列表 |

注意：

- `total_students` 当前等于 `assigned_students`。
- `accuracy`、`progress`、`completion_rate` 都是 0 到 1 的小数，前端展示百分比时需要乘以 100。

## 5. 题目统计 question_stats

`question_stats` 每一项代表一道题。

示例：

```json
{
  "question_id": 1,
  "question_pk": 1,
  "knowledge_point": "分数乘法",
  "question_type": "single_choice",
  "difficulty": "easy",
  "stem": "下列计算正确的是？",
  "options": [
    {
      "id": "A",
      "text": "1/2 × 1/3 = 1/6",
      "explanation": ""
    },
    {
      "id": "B",
      "text": "1/2 × 1/3 = 1/5",
      "explanation": ""
    }
  ],
  "correct_answer": ["A"],
  "total_answers": 42,
  "correct_count": 36,
  "wrong_count": 6,
  "accuracy": 0.8571,
  "option_stats": [
    {
      "option_id": "A",
      "option_text": "1/2 × 1/3 = 1/6",
      "count": 36,
      "is_correct": true
    },
    {
      "option_id": "B",
      "option_text": "1/2 × 1/3 = 1/5",
      "count": 6,
      "is_correct": false
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `question_id` | number | 题目 ID |
| `question_pk` | number | 题目主键，当前和 `question_id` 一致 |
| `knowledge_point` | string | 知识点 |
| `question_type` | string | 题型 |
| `difficulty` | string | 难度 |
| `stem` | string | 题干 |
| `options` | array | 题目选项 |
| `correct_answer` | string[] | 正确答案 ID 列表 |
| `total_answers` | number | 当前筛选范围内该题作答次数 |
| `correct_count` | number | 正确次数 |
| `wrong_count` | number | 错误次数 |
| `accuracy` | number | 正确率 |
| `option_stats` | array | 选项统计 |

## 6. 题型说明

当前后端支持的题型值：

| 值 | 含义 | option_stats |
|---|---|---|
| `single_choice` | 单选题 | 有 |
| `multiple_choice` | 多选题 | 有 |
| `judgment` | 判断题 | 有 |
| `blank` | 填空题 | 当前为空数组 |

第一版只统计选择题、判断题的选项选择次数。

填空题当前返回：

```json
{
  "option_stats": []
}
```

## 7. 选项统计 option_stats

`option_stats` 用于展示每个选项被学生选择的次数。

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `option_id` | string | 选项 ID，例如 `A`、`B` |
| `option_text` | string | 选项文本 |
| `count` | number | 当前筛选范围内选择该选项的人次 |
| `is_correct` | boolean | 是否为正确选项 |

前端展示建议：

- 用柱状图或横向进度条展示每个选项的 `count`。
- 正确选项可用绿色或正确标识。
- 错误选项可展示为普通颜色。
- 多选题中，一个学生选择多个选项时，每个被选选项都会分别计数。

## 8. 学生进度 student_progress

`student_progress` 每一项代表一个学生。

示例：

```json
{
  "student_no": "20300101",
  "student_name": "张三",
  "grade": "6",
  "class_name": "1",
  "answered_count": 8,
  "correct_count": 6,
  "accuracy": 0.75,
  "progress": 0.8,
  "status": "in_progress",
  "is_low_accuracy": false
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `student_no` | string | 学号 |
| `student_name` | string | 学生姓名 |
| `grade` | string | 年级 |
| `class_name` | string | 班级 |
| `answered_count` | number | 已答题数 |
| `correct_count` | number | 答对题数 |
| `accuracy` | number | 学生正确率 |
| `progress` | number | 学生完成进度 |
| `status` | string | 学生任务状态 |
| `is_low_accuracy` | boolean | 是否低正确率 |

`status` 取值：

| 值 | 含义 |
|---|---|
| `not_started` | 未开始 |
| `in_progress` | 进行中 |
| `completed` | 已完成 |

注意：

- 现在 `student_progress` 会包含未开始答题的学生。
- 未开始学生通常是 `answered_count = 0`、`progress = 0`、`status = "not_started"`。

## 9. 正确率最低题目 lowest_accuracy_questions

`lowest_accuracy_questions` 是正确率最低的题目列表。

结构与 `question_stats` 中的单个题目一致，也包含：

- 题干 `stem`
- 选项 `options`
- 正确答案 `correct_answer`
- 选项统计 `option_stats`
- 正确率 `accuracy`

前端可以直接复用题目统计卡片组件展示。

## 10. 前端页面建议

统计分析页建议包含：

1. 年级筛选器。
2. 班级筛选器。
3. 概览指标区：应参与、已提交、已完成、完成率、平均正确率、低正确率人数。
4. 题目统计列表：展示题干、知识点、难度、正确率、错误人数。
5. 题目详情弹窗或展开区：展示选项统计。
6. 学生进度表：展示姓名、学号、班级、完成进度、正确率、状态。
7. 正确率最低题目区域：帮助老师快速定位薄弱题。

## 11. 前端改动清单

前端需要改：

1. 调用分析接口时拼接 `grade` 和 `class_name` 查询参数。
2. 统计概览使用新的筛选后结果，不需要前端自行过滤。
3. 题目统计列表读取 `question_stats[*].stem`、`knowledge_point`、`difficulty`。
4. 选项统计读取 `question_stats[*].option_stats`。
5. 学生表格读取 `student_progress[*].grade` 和 `student_progress[*].class_name`。
6. 展示百分比时将小数乘以 100。
7. 对 `option_stats: []` 的填空题做空状态展示。

## 12. 错误处理

任务不存在时：

```json
{
  "detail": "Task not found"
}
```

前端处理建议：

- HTTP `404`：提示任务不存在或已删除。
- HTTP 非 `2xx`：读取 `detail` 并展示错误信息。
- `question_stats` 为空：展示暂无题目统计。
- `student_progress` 为空：展示当前筛选条件下暂无学生。
